## 阶段详情

### Phase 4: 主题基础设施

**目标：** 建立主题系统核心基础，实现主题配置、状态管理和切换逻辑

**依赖：** 无（继续 v1.0 代码库）

**需求：** THEME-01, THEME-02, THEME-04

**成功标准**（阶段完成时必须为真）：
1. 用户可以在 5 个预设主题之间切换
2. 主题切换时有平滑的视觉过渡（0.15-0.3s）
3. 主题状态在 Pinia Store 中正确管理

**计划：**
- [x] 04-01-PLAN.md — 定义主题类型系统和 CSS 变量基础（THEME-02, THEME-04）✓
- [x] 04-02-PLAN.md — 创建 Pinia Theme Store（THEME-01）✓
- [x] 04-03-PLAN.md — 创建 useTheme Composable（THEME-01, THEME-04）✓

### Phase 5: 主题集成

**目标：** 将主题系统应用到 UI 层，实现持久化和响应式适配

**依赖：** Phase 4

**需求：** THEME-03, THEME-05

**成功标准**（阶段完成时必须为真）：
1. 用户选择的主题在刷新页面后保持
2. 主题切换器在游戏头部显示，移动端布局正确
3. 主题切换在所有浏览器中一致工作

**计划：**
- [x] 05-01-PLAN.md — 创建主题切换器组件并实现 localStorage 持久化（THEME-03, THEME-05）✓
- [x] 05-02-PLAN.md — 迁移 UI 组件使用 CSS 变量（Tile.vue, GameHeader.vue, App.vue）（THEME-03, THEME-05）✓

### Phase 6: 性能优化

**目标：** 优化动画性能，使用 GPU 加速确保流畅度

**依赖：** 无（可独立于主题系统）

**需求：** PERF-01, PERF-02

**成功标准**（阶段完成时必须为真）：
1. 所有方块动画使用 transform 和 opacity
2. 动画不触发布局重排（width/height/top/left）
3. 动画元素使用 will-change 提示（仅动画期间）

**计划：**
- [x] 06-01-PLAN.md — 录制性能基线并审查现有动画实现（PERF-01, PERF-02）✓
- [x] 06-02-PLAN.md — 优化动画实现，使用 GPU 加速和动态 will-change 管理（PERF-01, PERF-02）✓

### Phase 7: 性能验证

**目标：** 量化优化效果，验证 60fps 目标达成

**依赖：** Phase 6

**需求：** PERF-03

**成功标准**（阶段完成时必须为真）：
1. 游戏动画在 Chrome DevTools 中显示稳定 60fps
2. Long Tasks < 50ms（无主线程阻塞）
3. 低端设备（3 年前手机）测试流畅
