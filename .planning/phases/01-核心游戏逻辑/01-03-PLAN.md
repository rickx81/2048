  <behavior>
    ## 游戏结束检测（GAME-07）

    **游戏结束条件：** 网格填满且无法进行任何合并

    **非游戏结束的情况：**
    - 网格有空位 → false
    - 网格填满但可以水平合并 → false
    - 网格填满但可以垂直合并 → false

    **游戏结束的情况：**
    - 网格填满（无 0）且所有相邻方块都不同

    **示例：**
    ```
    [2, 4, 8, 16]
    [4, 2, 16, 8]   → false（有空位）
    [0, 8, 4, 2]
    [16, 8, 4, 2]

    [2, 4, 8, 16]
    [4, 2, 16, 8]   → false（可以垂直合并：2-2, 4-4, 8-8, 16-16）
    [2, 4, 8, 16]
    [4, 2, 16, 8]

    [2, 4, 8, 16]
    [4, 2, 32, 64]  → false（可以水平合并：第一列 2-2, 4-4）
    [2, 128, 256, 512]
    [2, 4, 8, 16]

    [2, 4, 8, 16]
    [4, 2, 32, 64]  → true（无法进行任何合并）
    [2, 128, 256, 512]
    [4, 8, 16, 32]
    ```

    ## 游戏胜利检测（GAME-08）

    **胜利条件：** 网格中存在 2048 方块

    **示例：**
    ```
    [2, 4, 8, 16]
    [32, 64, 128, 256]   → false（无 2048）
    [512, 1024, 512, 1024]
    [2048, 4096, 8192, 16384]

    [2, 4, 8, 16]
    [32, 64, 128, 256]   → true（有 2048）
    [512, 1024, 2048, 4096]
    [8192, 16384, 32768, 65536]
    ```

    **注意：** 达到 2048 后游戏不强制结束，用户可以继续玩

    ## 有效移动检测（辅助函数）

    **hasValidMoves(grid)**: 判断是否还有有效移动

    实现：
    - 如果有空位 → true
    - 如果可以水平合并 → true
    - 如果可以垂直合并 → true
    - 否则 → false

    **性能优化：** 一旦发现有效移动立即返回 true，避免不必要的遍历
  </behavior>
  <implementation>
  在 src/core/game.ts 中实现以下函数（采用 TDD RED→GREEN→REFACTOR 循环）：

  ## 1. 游戏胜利检测

  **isGameWon(grid)**: 判断游戏是否胜利
  ```typescript
  export function isGameWon(grid: Grid): boolean
  ```
  实现：
  - 遍历网格，查找是否存在 2048
  - 找到即返回 true
  - 遍历完仍未找到返回 false

  ## 2. 辅助检测函数

  **hasEmptyCell(grid)**: 判断是否有空位
  ```typescript
  function hasEmptyCell(grid: Grid): boolean
  ```
  实现：
  - 遍历网格，查找是否存在 0
  - 找到即返回 true
  - 遍历完仍未找到返回 false

  **canMerge(grid)**: 判断是否可以合并（水平或垂直）
  ```typescript
  function canMerge(grid: Grid): boolean
  ```
  实现：
  - 水平检查：遍历每一行，检查相邻元素是否相同
  - 垂直检查：遍历每一列，检查相邻元素是否相同
  - 发现可合并的相邻元素即返回 true
  - 检查完所有行列仍未发现返回 false

  ## 3. 游戏结束检测

  **hasValidMoves(grid)**: 判断是否还有有效移动
  ```typescript
  export function hasValidMoves(grid: Grid): boolean
  ```
  实现：
  - 调用 hasEmptyCell(grid)，有空位则返回 true
  - 调用 canMerge(grid)，可合并则返回 true
  - 都不满足则返回 false

  **isGameOver(grid)**: 判断游戏是否结束
  ```typescript
  export function isGameOver(grid: Grid): boolean
  ```
  实现：
  - 调用 hasValidMoves(grid)
  - 取反结果（无有效移动 = 游戏结束）
  - 返回 !hasValidMoves(grid)

  ## 实现要点

  - 所有函数都是纯函数
  - 性能优化：短路求值，一旦确定结果立即返回
  - 代码复用：isGameOver 复用 hasValidMoves
  - 边界处理：正确处理空网格、全满网格、单元素网格
  - 逻辑清晰：每个函数单一职责，易于测试
  </implementation>
