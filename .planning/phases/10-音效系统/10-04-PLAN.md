根据 10-RESEARCH.md 和 useAudio.ts 的配置，sprite.mp3 需要包含以下音效：

| 音效 | 开始时间 | 持续时间 | 用途 |

| merge | 200ms | 300ms | 数字合并时播放 |

| lose | 2700ms | 1500ms | 游戏失败时播放 |

**注意：** 时间戳需要与 `src/composables/useAudio.ts` 中的 SPRITE_DATA 配置完全一致。

## 音频精灵制作指南

### 方法 1：使用 audiosprite 工具（推荐）

```bash
# 安装 audiosprite
npm install -g audiosprite

# 准备 4 个独立的音效文件
# 将它们放在同一个目录下（如 temp_sounds/）

# 生成音频精灵
audiosprite \
  --output sprite \
  --format mp3 \
  --bitrate 128 \
  --samplerate 44100 \
  --gap 50 \
  temp_sounds/move.mp3 \
  temp_sounds/merge.mp3 \
  temp_sounds/win.mp3 \
  temp_sounds/lose.mp3

# 生成的文件：
# - sprite.mp3（合并后的音频）
# - sprite.json（时间戳定义）

# 将 sprite.mp3 移动到项目目录
mkdir -p public/sounds
mv sprite.mp3 public/sounds/sprite.mp3
```

### 方法 2：使用在线工具

- [AudioSprite Generator](https://www.leshylabs.com/apps/sstool/) - 上传多个音频文件，生成 sprite
- 下载生成的 sprite.mp3 文件

### 方法 3：使用音频编辑器手动合并

1. 在 Audacity 中按顺序导入 4 个音效文件
2. 添加静音间隔（每个音效之间 50ms）
3. 导出为 MP3 文件
4. 手动调整 useAudio.ts 中的 SPRITE_DATA 时间戳以匹配实际音频
