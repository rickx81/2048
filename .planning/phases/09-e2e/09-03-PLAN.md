**测试结构：**

1. **test.describe('键盘方向键控制')**
   - `'ArrowRight 键触发向右移动'` — 记录移动前 grid state，按 ArrowRight，验证 grid 变化
   - `'ArrowLeft 键触发向左移动'` — 同上，ArrowLeft
   - `'ArrowUp 键触发向上移动'` — 同上，ArrowUp
   - `'ArrowDown 键触发向下移动'` — 同上，ArrowDown
   - `'WASD 键也能控制方向'` — 验证 W/A/S/D 键也生效（因为 useGameControls.ts 支持 WASD）
     - 按 'a' 键，验证与 ArrowLeft 相同的移动效果

   实现每个方向键测试的通用模式：
   ```typescript
   async function verifyDirectionMove(gamePage: GamePage, key: string) {
     const gridBefore = await gamePage.getGridState()
     await gamePage.pressKey(key as any)
     const gridAfter = await gamePage.getGridState()
     // 至少有一个格子的值应该变化（初始状态总是可以移动的）
     const changed = JSON.stringify(gridBefore) !== JSON.stringify(gridAfter)
     expect(changed).toBe(true)
   }
   ```

2. **test.describe('触摸滑动控制')**
   - `'向右滑动触发移动'` — 使用 gamePage.swipe('right')，验证 grid 变化
   - `'向左滑动触发移动'` — 使用 gamePage.swipe('left')，验证 grid 变化
   - `'向上滑动触发移动'` — 使用 gamePage.swipe('up')，验证 grid 变化
   - `'向下滑动触发移动'` — 使用 gamePage.swipe('down')，验证 grid 变化
   - `'短距离滑动不触发移动'` — 使用 gamePage.swipe('right', 10)（低于 SWIPE_THRESHOLD=50），验证 grid 不变

   注意：触摸滑动测试使用鼠标模拟（mouse.move + mouse.down + mouse.up），因为 Playwright 的 touchscreen API 不支持滑动序列。GamePage.swipe() 方法已在 09-01 中使用 mouse API 实现。

3. **test.describe('游戏状态影响控制')**
   - `'游戏结束后方向键不响应'` — 使用 localStorage 注入一个无法继续的棋盘状态，触发游戏结束，再按方向键验证无变化
   - `'游戏胜利后继续游戏可以操作'` — 注入接近胜利的状态，触发胜利，点击继续游戏，验证可以继续移动

**重要细节：**
- 所有测试独立运行，不依赖执行顺序
- 每个测试使用新的 gamePage fixture（自动执行 goto）
- 不使用 waitForTimeout，依赖 Playwright 自动等待
- 触摸测试使用 mouse API 模拟（与 GamePage.swipe 实现一致）
- 键盘测试直接使用 page.keyboard.press
