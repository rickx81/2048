### Phase 10: 音效系统

**目标**: 用户在游戏过程中获得音频反馈，提升游戏体验

**依赖**: Phase 9

**需求**: AUDIO-01, AUDIO-02, AUDIO-03, AUDIO-04, AUDIO-05, AUDIO-06, AUDIO-07

**成功标准**（必须为真）：
1. 用户在移动方块时能听到移动音效
2. 用户在数字合并时能听到合并音效
3. 用户在游戏胜利时能听到胜利音效
4. 用户在游戏失败时能听到失败音效
5. 用户可以通过 UI 控制调节音效音量
6. 用户可以通过 UI 控制静音所有音效
7. 音效设置（音量、静音）持久化到 localStorage，刷新页面后保持

**计划**: 3 个计划

**Plans**:
- [x] 10-01-PLAN.md — Howler.js 依赖 + 音效基础设施（composable + store）
- [x] 10-02-PLAN.md — 音效资源 + 音量控制 UI 组件
- [ ] 10-03-PLAN.md — 游戏流程集成 + 浏览器自动播放策略处理

**UI 提示**: yes

**需求**: BUILD-01, BUILD-02, BUILD-03, BUILD-04, BUILD-05, BUILD-06
