**Composable 结构：**
参考 `src/composables/useGameControls.ts` 的模式，返回响应式状态和方法：

```typescript
import { watch } from 'vue'
import { useThemeStore } from '@/stores/theme'
import type { ThemeConfig } from '@/core/theme'

/**
 * 主题 Composable
 * 提供统一的主题访问接口，自动处理 DOM 副作用
 */
export function useTheme() {
  const store = useThemeStore()

  // 监听主题变化，应用到 DOM
  watch(
    () => store.currentThemeId,
    (newThemeId) => {
      // 设置 data-theme 属性，激活对应的 CSS 变量
      document.documentElement.dataset.theme = newThemeId
    },
    { immediate: true } // 立即执行以初始化
  )

  return {
    /** 当前主题 ID（响应式） */
    currentThemeId: store.currentThemeId,
    /** 当前主题配置对象（响应式） */
    currentTheme: store.currentTheme,
    /** 设置主题方法 */
    setTheme: store.setTheme
  }
}
```

**关键规则：**
- 使用 `watch` 监听 `currentThemeId` 变化
- 设置 `immediate: true` 确保初始化时立即应用主题
- 通过 `document.documentElement.dataset.theme` 设置 data 属性
- 返回 Store 的状态和方法（转发模式）
- **不包含持久化逻辑**（localStorage 在 Phase 5 添加）
- 添加完整的 JSDoc 注释（中文）

**与现有 useGameControls.ts 的区别：**
- useGameControls 监听键盘和触摸事件
- useTheme 监听主题 ID 变化并操作 DOM
- 两者都封装副作用，提供清晰的 API

**DOM 操作说明：**
- `document.documentElement` = `<html>` 元素
- `dataset.theme` 设置 `data-theme` 属性
- CSS 中的 `[data-theme='neon']` 选择器会匹配该属性
- CSS 变量会根据 data 属性值自动切换
