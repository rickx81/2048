  <name>任务 1：准备音频精灵文件</name>
  <files>public/sounds/sprite.mp3</files>
  <what-built>
    音频精灵文件（sprite.mp3）已添加到 public/sounds/ 目录。

    此文件包含 4 个音效：
    - move（移动音效）：0ms-150ms
    - merge（合并音效）：200ms-500ms
    - win（胜利音效）：600ms-2600ms
    - lose（失败音效）：2700ms-4200ms
  </what-built>
  <how-to-verify>
    1. 获取或制作 4 个音效文件（move.mp3, merge.mp3, win.mp3, lose.mp3）
       - 推荐来源：Freesound.org 或 OpenGameArt.org 的自由版权音效
       - 或使用 Web Audio API 合成简单音效

    2. 使用音频精灵工具合并音效：
       ```bash
       # 安装工具（如果未安装）
       npm install -g audiosprite

       # 合并音效
       audiosprite \
         --output public/sounds/sprite \
         --format mp3 \
         move.mp3 merge.mp3 win.mp3 lose.mp3

       # 将生成的 sprite.mp3 移动到正确位置
       mv sprite.mp3 public/sounds/sprite.mp3
       ```

       或使用在线工具：https://www.codeandweb.com/audiokit

    3. 确认文件存在：
       - public/sounds/sprite.mp3 文件大小应 > 1KB
       - 使用音频播放器测试可正常播放

    4. 如果跳过此任务：
       - 音效系统将在文件加载时静默失败
       - 控制台会显示音效加载错误（但不影响游戏功能）
  </how-to-verify>
  <resume-signal>输入 "音效文件已添加" 或 "跳过（稍后添加）"</resume-signal>
</task>

       - 滑块宽度：120px
       - 过渡动画：0.15s ease

    5. **可访问性**（参考 UI-SPEC.md 第 218-223 行）：
       - 按钮 title 属性："音量控制"（未静音）/ "已静音"（静音）

       - 滑块 aria-valuemin="0" 和 aria-valuemax="100"
       - 静音状态通过 aria-pressed 传达
